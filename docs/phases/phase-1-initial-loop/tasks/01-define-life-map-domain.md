# Define the life-map domain

Status: in-progress

Owner: primary-agent

## Outcome

Define the plain data model and tested rules for users, islands, activities, nodes, cross-island placement, rocks, sinking, and resurfacing.

## Done when

The model represents the established product behavior, including one summarized node and point per island per day, mixed positive and harmful days, repeated check-in merging, current-day-only logging, five-point positive growth milestones, five-negative-point rocks, one-rock-to-one-lifetime-point sinking weight, resurfacing when lifetime points exceed rocks, transparent corrections, and a five-minute undo window. Unresolved constants are explicit, and core rule examples pass automated tests without depending on the renderer.
